## How signing works

GitHub does not have a "sign" button in the literal sense. The signing mechanism is operationally:

1. You click **["Sign Acknowledgment" → opens a pre-filled GitHub Issue](https://github.com/ametemre/yrenn-hybrid/issues/new?template=sovereign-rights-acknowledgment.yml)**.
2. The issue form asks for your handle, optional affiliation, intended use, and four explicit acknowledgments (one per clause above).
3. You submit. GitHub stamps your handle and the timestamp on the resulting issue.
4. The issue is your **public, durable, GitHub-identity-bound** signature.
5. The label `sovereign-rights-ack` is auto-applied; the maintainer does not need to take action for the signature to be valid.

The signature is **valid the moment you submit**. There is no review or approval step. The signature is **not revocable retroactively** — that is the point; the GitHub timestamp is a public commitment.
